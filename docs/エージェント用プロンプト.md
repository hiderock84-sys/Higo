# ひごのいえ UI 作業用プロンプト（コピー用）

以下を Cursor / Cloud Agent に貼り付けて使ってください。

---

## プロンプト本文

```
ひごのいえ（https://higonoie.com/）のランディングUIを修正してください。

【正本】
- ルート index.html
- public/static/style.css
- public/static/hero-variants.css
hidonoie/ や higono-ie/ は触らない。ルートだけが本番のソース。

【作業前】
git checkout HEAD -- index.html public/static/style.css public/static/hero-variants.css
npm run verify:landing
を実行し、現状を確認してから編集する。

【絶対に守ること】
1. style.css を 5500 行台の短いファイルに差し替えない（正は 7800 行以上。concerns-section / glass-btn / hero-actions--quad の CSS が含まれること）
2. index.html に #page-menu・「ページメニュー」を復活させない
3. <body> に is-landing / hero-variant-bottom を直書きしない（JS の clearPageViewState で制御）
4. CSS 変更時は index.html の ?v= キャッシュキーも更新する
5. 編集後は必ず npm run verify:landing && npm run build が通ること
6. 通ったら commit → push → npm run deploy:prod

【維持すべきUI】
- 表紙: ヒーローコピーは上、4つのガラスボタン（無料相談・ブログ・24h・家族）は下。空の空を作らない
- お悩み: #concerns はカードグリッド（アイコンと文言が横並びカード）
- フッター: 余白コンパクト、スマホでナビ2カラム
- モバイルメニュー: 写真付きカード、お問い合わせブロックなし
- 単体ページ /program 等: is-landing が残らず1セクションだけ表示

【完了報告に含めること】
- verify:landing の結果
- style.css の行数
- デプロイした ?v= の値
```

---

## なぜ戻るのか（根本原因）

| 原因 | 内容 |
|------|------|
| CSS 本体の欠落 | 一度 `style.css` から約2500行削除され、HTML だけ新しくても見た目が崩れる |
| 正本の取り違え | 古い `index.html`（ページメニューあり）を編集・デプロイしてしまう |
| body クラス直書き | `is-landing` が全URLに残り、単体ページと表紙のCSSが競合 |
| 検証なしデプロイ | `npm run verify:landing` 前に push すると欠落に気づかない |

回帰テストは `scripts/verify-landing-ui-regression.mjs` に集約済みです。
